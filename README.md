# BookDB — conversational hybrid book recommender

A small, runnable book recommender that fuses **three signals** and is driven by
a **natural-language query layer**. You type "something like Dune but lighter" or
"a short mystery set in Japan"; it parses your intent, ranks books with a hybrid
of collaborative filtering and semantic search, and explains every pick.

🔗 **Live demo:** https://andreaisabelmontana.github.io/bookdb-discovery/
(the page is a UI; the engine described here is the real Python in `discovery/`.)

## The three signals

| signal | how | code |
|---|---|---|
| **Collaborative filtering** | item-based CF — mean-centered cosine over a user×book ratings matrix; score candidates by similarity-weighted votes from your seed books | `discovery/collab.py` |
| **Semantic search** | TF-IDF (1–2 grams) + **LSA** (TruncatedSVD) over each book's title + genres + tags + description; cosine similarity to a free-text query or a seed book | `discovery/semantic.py` |
| **Hybrid fusion** | **Reciprocal Rank Fusion** (default) of the CF and semantic rankings, weighted by `alpha`; also supports a weighted-score blend | `discovery/hybrid.py` |

RRF rewards a book ranked highly by *either* signal, so the fused list captures
the **union** of what co-rating and content each find — which is why it beats
either signal alone (numbers below).

## The conversational layer

`discovery/query.py` is a transparent, rule/keyword intent parser — **no large
language model**. It turns free text into a structured `Intent`:

- **seed book** from `like / similar to X` (regex + fuzzy title match against the catalog)
- **genre / mood / setting** filters from a cue vocabulary (`thriller`, `cozy`, `japan`, …)
- **mood modifiers**: `but lighter` → prefer `cozy/warm`, **avoid** `sad/eerie`; `but darker` → the reverse
- **length**: `short`, `not too long` → `short`; `epic`, `long` → `long`

Every extracted field is traceable to the phrase that triggered it (`Intent.explanation`).
`discovery/engine.py` runs the intent through the hybrid ranker and attaches a
plain-English reason to each recommendation.

> Honest scope: "AI" here means TF-IDF/LSA semantic search + keyword/rule intent
> parsing + CF fusion. There is no LLM and no neural net. The ranker is real and
> tested; the data is **synthetic** (see below).

## Data

`data/books.csv` — **64 books** (title, author, genres, tags, length, year, description),
a hand-written catalog of widely-known titles.
`data/ratings.csv` — **5,475 synthetic ratings** from 360 simulated readers,
generated by `data/make_ratings.py` (committed, deterministic). Each simulated
reader has a genre/mood taste plus a few idiosyncratic favourites, so the ratings
carry real item–item co-rating structure for CF **and** are deliberately *not*
redundant with the book text — that complementarity is what lets the hybrid win.
Ratings are clearly labeled synthetic; no real user data is used.

## Evaluation — hybrid vs. single signals

Leave-one-out: for each reader who liked ≥ 6 books, hold one liked book out, seed
the recommender with the rest, and check whether the held-out book lands in the
top-K. Reported as **Hit-Rate@10** (mean over 5 held-out draws) and **MRR**.
Run it yourself with `python demo.py`. Real output on the committed data:

**Full data** (n = 357 readers, `alpha = 0.6`):

| ranker | Hit-Rate@10 | MRR |
|---|---|---|
| collaborative-filtering only | 0.673 | 0.300 |
| semantic-search only | 0.550 | 0.199 |
| **hybrid (RRF)** | **0.680** | 0.283 |

→ the hybrid beats **both** single signals on Hit-Rate@10.

**Sparse data** — CF trained on only 15% of the ratings (a new/small platform):

| ranker | Hit-Rate@10 |
|---|---|
| collaborative-filtering only | 0.342 |
| semantic-search only | 0.515 |
| **hybrid (RRF)** | **0.473** |

→ CF collapses when starved of data, but the content channel keeps the hybrid
**+0.13 ahead of CF** and robust. The hybrid is the only ranker that is never the worst.

## Example queries (real `demo.py` output)

```
QUERY: "a dreamlike literary novel set in Japan"
  parsed intent: genres=[literary fiction]  tags=[japan]
  1. Kafka on the Shore — Haruki Murakami        [CF 0.00  semantic 1.00]
  2. The Wind-Up Bird Chronicle — Haruki Murakami [CF 0.00  semantic 0.91]
  3. Convenience Store Woman — Sayaka Murata      [CF 0.00  semantic 0.70]

QUERY: "something like Never Let Me Go but lighter"
  parsed intent: seed=Never Let Me Go  prefer=[cozy,warm]  avoid=[sad,eerie]
  1. The Remains of the Day — Kazuo Ishiguro   [CF 0.67  semantic 0.35]
  2. A Gentleman in Moscow — Amor Towles        [CF 1.00  semantic 0.27]
  3. Normal People — Sally Rooney               [CF 0.52  semantic 0.37]
  (the sad books that 'like Never Let Me Go' alone surfaces — Klara and the Sun,
   A Little Life — are dropped by the 'lighter' constraint)
```

## Layout

```
discovery/
  data.py        catalog + ratings loading, fuzzy title lookup
  collab.py      item-based collaborative filtering
  semantic.py    TF-IDF + LSA semantic index
  hybrid.py      RRF / weighted fusion ranker
  query.py       natural-language intent parser
  engine.py      end-to-end: intent -> hybrid -> explanations
  evaluate.py    leave-one-out hit-rate / MRR (full + sparse)
data/            books.csv, ratings.csv, make_ratings.py
tests/           query parser, semantic, collab, hybrid-eval, constraints
demo.py          runnable demo
```

## Tests cover

- the parser extracts the right **seed book + filters** from example queries
- semantic search returns **topically-correct** books (Japanese mysteries, cozy fantasy, space survival)
- the **hybrid beats CF-only and semantic-only** on held-out Hit-Rate@10
- `like X but <constraint>` **respects the constraint** ("but lighter" drops sad/eerie books)
